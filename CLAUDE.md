# Engineering Principles

## KISS
Prefer the simplest solution.

## YAGNI
Build only current requirements.

## Rule of Three
Don't abstract before three occurrences.

## DRY
Remove duplication only after patterns emerge.

## Separation of Concerns
Keep UI, logic, and data separate.

## SRP
One responsibility per component.

## Default
Simple > Clever.
Concrete > Generic.
Boring > Fancy.

Before writing code:
- Can this be simpler?
- Am I solving a future problem?
- Is this abstraction justified?

After writing code:
- Remove unnecessary complexity.
- Merge files if possible.
- Prefer fewer moving parts.
